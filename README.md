## 重要な注意事項

### セキュリティ
- **データベース接続情報は絶対にGit管理しない**
- SQLインジェクション対策は必須（プリペアドステートメント使用）
- POST処理後は必ずリダイレクト（二重送信防止）
- JavaScriptはクライアント側の処理のみ（サーバー側の検証は必須）

### 開発ルール
- ファイルパスは`__DIR__`または`BASE_PATH`を使用（`../`は原則使わない）
- SQL文は1か所にまとめる（直接DBを触らない）
- エラーハンドリングを必ず実装
- コードレビューを実施
